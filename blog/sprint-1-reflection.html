Branch, Pull, Merge

What is Master?

Master is the main Repo that contains all of the approved features that have been worked on in other branches. It is what contains the final deployable product.

Why create a Branch?

Creating a branch allows you to make any changes or additions to features without affecting the master. This means that if something breaks, it will not affect the most important part of the repo. It also allows for multiple people to work on features and bugs at one time without impacting eachothers work or slowing productivity in any way.

GitHub Fork & Clone

What are some examples of when you would fork?

Perhaps you are looking at an open source project with lots of contributors. In order to make changes without affecting or potentially breaking the main product, it is best to create a fork. This allows you to make all the changes you want to the master without affecting the actual project because you have forked it into your own personal repo.

Write your own step-by-step fork and clone instructions. Describe what you did.

  1. When you want to Fork a repo, in GitHub click the Fork button located on the right of the screen.
  2. Select which developer account you want to Fork the repo into.
  3. Clone the forked repo onto your local computer.
  4. In GitBash, make any branches and changes to files/features that you need.
  5. When finished with your feature or fix. Stage (git add -A) and commit (git commit -m "message") the files and push (git push origin branchName) them onto the forked Repo.
  6. Submit a Pull Request to the owner or head developer of the origin repo. If approved, this will merge the changes you made on you repo with the origin.

  7. Delete any branches that are not needed after the merge is complete.

Thinking like a programmer

Describe the process for solving problems

The process of solving a problem involves making a calculated, strategic apporach to overcoming a problem rather than blindly hacking away hoping that a solution will stick. The approach involves understanding the problem that needs to be solved, breaking that problem down into sub problems, and creating a plan or roadmap to fixing said problem. Effective problem solving also includes refactoring and reflection once a problem has been solved so that it is solved well and that the solution has been understood and learned from.

Will you follow that process? How?

I want to train myself to make this approach a habit. This requires me to use these methods as often as possible in any problem solving situation, even if I know its a google search away. Although there is nothing wrong with quickly googling an answer. Developing the habit of going through the process to solve a problem is the best way for the habit to stick.

What did you learn about flipped classroom?

I learnt that a Flipped Classroom encourages an Intentional Learning Culture that sees instruction move from the group to the individual. Rather than a Teacher being the source of information and answers, it is passed off to the students and the Teacher acts as more of a guide for the students. When moved back into the group space, it allows for students to practice concepts and ideas through intentional learning exercises compared to simply having concepts explained to them in a traditional classroom setting.
