Do you think you would prefer using Unit Testing or Test Driven Development? Why?

I think in all honesty I would use both. Test Driven Development is a good way to establish a working structure of a program before getting to intesely into details not knowing if something fully works or not. Unit testing is a great way to apply test driven development by structuring your unit testing in the same way you plan to structure your program. I see them working better together than each individually on their own.

How could these techniques be used together to make code that is reliable?

Seeing as Testing Driven Development relies on testing the structural aspects of a program, Unit testing fits perfectly into this. It breaks down the structure of a program into smaller functions. This way you can plan your structure based on the unit tests you need to pass before you even write the main flesh of the code.

Was your initial response to the idea of testing, "That sounds useful" or "That sounds like a hassle"? Why?

I understand why its there, and ideally my code would magically work without the need of in depth testing. But I know that is not the case and testing is absolutely needed in order to either fix a bug or optimize a feature. I can see use for it beyond simply getting a pass or fail on a feature. To me it seems like it will be a way to look into my code and further optimize or refactor or tidy unneccary elements.
