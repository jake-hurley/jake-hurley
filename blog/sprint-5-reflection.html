My Reflections - Sprint Five

Minesweeper

What was the hardest moment of this challenge for you?

I think for me the hardest part of this challenge was coding something around a pre-existing structure. It forces you to be a bit smarter and more calculated with the things you add because you have to think about if it will work well alongside the already written code. Once I had a look around and understood what was doing what and what functions were being called, I felt much more comfortable with what I was trying to do.

Did you have any blocks during this challenge? How did you overcome them?

I had quite a few blocks especially at the start. I think persistence firstly was an important factor, but also making sure I am thinking about everything the code I am writing is trying to do. In most cases a couple of rounds of Rubber Duckey method helped me understand where problems were happening. Ontop of that, finding some internet sources around specific things like random number generation and audio made life much easier.

What learnings are you going to take away from this challenge?

I think an important element I need to focus on in other challenges is making sure I am not overlooking the basic solutions. There was a problem I faced in the challenge that was solved by changing where a line of code was placed. But because I was looking for more complex methods to solve the problem, when I realised that I had overlooked a really basic solution I got frustrated. So making sure I remember the basic checklist to follow next time is going to save a lot of frustration.

If _you_ were responsible for curriculum development at Dev Academy, would you change this challenge in any way? How? Why?

I think it was good in the sense that it didn't hand feed the answers and required some more critical thought around what needed to be done in order to complete the challenge. But at times it seemed a bit unclear on first read through of what needed to be done to get things working.

JavaScript Carnival

Inflate the Unicorn

Did you Learn anything new during this exercise?

Yes, there were a couple of new features that I had not known about such as the onclick method in Html. I also learnt how important DOM selectors are when it comes to working with multiple objects that have the same tag or class.

Did you encounter issues completing any ofhte steps of the exercise? How did you resolve this?

The main issue I encountered was getting the one function to work for all 3 unicorns without changing the other unicorns in the process. The main problem was that I was trying to make one function do everything and it was causing problems. After an hour or so of frustration, the solution came down to a friend reminding me of the idea that a function should not have more than one purpose. After I realised this it was a pretty quick process to split up the section of the function that updates the counts and that made everything work smoothly.

If you were to replace the unicorn with another mystical animal of your choice, what would it be?

Probably just a Dragon or something, they are pretty cool.

Whack a Mole

What was the hardest part of this exercise?

I think getting used to changing attributes of Html items was the concept that took a little bit longer to grasp. Once I got more comfortable with that section, the challenge became easier to complete.

Can you think of any different ways you could have accomplished the same outcome?

Not at this stage. Not because my code is amazing and not worth changing. But more so just because I am still pretty elementary with my approach and havent witnessed multiple ways of solving the same problem yet. I think when I get a bit more exposure to other peers and colleagues ways of doing things, I will start to see multiple ways around problems I am trying to solve.

Dress the Clown

How did you find the process of Refactoring your code?

Frustrating. It felt like I was starting the whole challenge all over again. Refactoring is hard because in many cases it causes a lot of the other code to need to be changed. To me it suggests that I might need to plan things a bit better initially instead of getting something working with the knowledge its going to have to be refactored.

Would you do anything differently if you were to start this challenge again?

Based on my answer to the previous question, I think I need to think harder about how to make something that doesnt require a massive amount of refactoring at the end.

What piece of code are you proudest of from this exercise?

I think I am pretty proud of getting the up down arrows working. When I started to work on that section, I was a bit intimidated with it. There was a little bit of doubt about my ability to get it done. So when I overcame the challenge I was really happy!
